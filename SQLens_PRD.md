## 1. 产品概述

### 1.1 产品简介

**SQLens**（SQL + Lens，意为"SQL 注入的透视镜"）是一款基于 Electron 的桌面端 SQL 注入分析工具。它不修改 SQLMap 源码，而是通过调用 `sqlmapapi` 的 REST API 实现全图形化操作，并引入大模型（默认 DeepSeek）实现全流程自动化 SQL 注入检测。

### 1.2 一句话定位

> **SQLens = 图形化 SQLMap 外壳 + AI 全自动注入分析**

### 1.3 核心价值

| 价值 | 说明 |
|------|------|
| **零命令行** | 所有 sqlmap 参数图形化配置，无需记忆命令 |
| **一键全自动** | 粘贴 URL/数据包 → AI 自动分析 → 自动扫描 → 自动出报告 |
| **AI 辅助** | 智能解读日志、推荐参数、生成修复建议 |
| **开箱即用** | 单文件安装包，自动管理 sqlmapapi 进程 |

## 3. 产品定位与差异化

### 3.1 竞品格局

| 产品 | 类型 | 状态 | 缺点 |
|------|------|------|------|
| Havij | 桌面 GUI | ❌ 停更10年 | 已过时，无法绕过 WAF |
| Pangolin | 桌面 GUI | ❌ 停更 | Windows only，功能有限 |
| jSQL Injection | 桌面 GUI (Java) | ✅ 活跃 | 独立引擎不如 SQLMap 强大 |
| sqlmapWebUI | Web GUI | ✅ 活跃 | 需要部署服务端，非桌面端 |
| SQLI Striker | Web GUI | ✅ 活跃 | 需要 Node 环境 |
| **SQLens** | **桌面 GUI (Electron)** | **🚀 新建** | **AI 全自动 + 全图形化** |

### 3.2 核心差异化优势

| # | 优势 | 说明 | 竞品状态 |
|---|------|------|---------|
| 1 | 🤖 AI 全流程自动化 | 从分析到报告，AI 全程驱动 | 竞品均无 |
| 2 | 🖥️ 全参数图形化 | sqlmap 所有常用命令 GUI 化 | 部分竞品不全 |
| 3 | 💬 AI 实时解读 | 扫描日志 AI 实时翻译成大白话 | 竞品均无 |
| 4 | 🛡️ 开发者模式 | 给出修复建议代码 | 竞品均无 |
| 5 | 📦 单文件开箱即用 | 自动管理 sqlmapapi | 竞品需手动配置 |

### 4.2 技术栈明细

| 层级 | 技术 | 选型理由 |
|------|------|---------|

| 前端框架 | **React 18** | 组件化，生态丰富 |

| AI API | **DeepSeek API** (默认，可自定义切换) | 效果好，价格低 |

| 自动更新 | **electron-updater** (GitHub Releases) | 启动时自动检测更新 |
| 内置运行时 | **嵌入式 Python** + **sqlmap 源码** | 用户双击 exe 即用，零依赖 |

### 4.3 sqlmapapi 交互流程

```
sqlmapapi REST API 调用链路:

1. 启动管理
   └─ Electron 自动执行: python sqlmapapi.py -s -H 127.0.0.1 -p 8775

2. 创建扫描任务
   └─ POST /task/new
      → 返回: {"success": true, "taskid": "xxx"}

3. 设置任务参数
   └─ POST /option/{taskid}/set
      → Body: {"url": "http://example.com?id=1", "level": 2, ...}

4. 启动扫描
   └─ POST /scan/{taskid}/start
      → 返回: {"success": true, "engineid": "yyy"}

5. 轮询状态
   └─ GET /scan/{taskid}/status
      → 返回: {"status": "running", "data": [日志条目...]}

6. 获取结果
   └─ GET /scan/{taskid}/data
      → 返回: {"data": [注入点信息, 枚举结果...]}

7. 停止任务
   └─ GET /scan/{taskid}/stop
```

### 5.3 自然语言输入（核心差异化功能）

用户可以通过底部聊天框，用中文自然语言指挥 SQLens 工作。AI 理解意图后自动执行。

**典型对话场景：**

| 用户说 | AI 理解并执行 |
|--------|-------------|
| _"扫一下这个网站 http://xxx.com?id=1"_ | 自动分析参数 → 配置 sqlmap → 开始扫描 |
| _"帮我看看有哪些数据库"_ | 发现注入后 → 自动执行 `--dbs` 枚举 |
| _"绕过 WAF，用高等级扫"_ | 调整参数 level=3，推荐 tamper 脚本，重新扫描 |
| _"把 users 表的数据导出来"_ | 枚举 users 表列 → dump 数据 → 展示在表格 |
| _"生成报告"_ | 汇总扫描结果 → 生成结构化报告 |
| _"这个漏洞怎么修"_ | 根据数据库类型 → 给出对应语言修复代码示例 |

**界面位置：** 应用底部固定聊天输入框，类似 ChatGPT 布局。

**两种模式：**
- **对话模式** — 用户打字指挥，AI 执行
- **静默模式** — 隐藏聊天框，纯 GUI 操作

### 6.2 AI Prompt 设计（概要）


## 8. 非功能需求

### 8.1 性能

| 指标 | 要求 |
|------|------|
| 应用启动时间 | ≤ 3 秒 |
| sqlmapapi 自动启动 | ≤ 5 秒 |
| 日志轮询间隔 | 2 秒 |
| 安装包大小 | 无严格限制（含嵌入式 Python + sqlmap） |
| 内存占用 | ≤ 200MB（空闲状态），≤ 500MB（扫描中） |

### 8.2 兼容性

| 平台 | 支持 |
|------|------|
| Windows 10/11 | ✅ |
| macOS 12+ | ✅ |
| Linux (Ubuntu/Debian) | ✅ |

### 8.3 安全性

- API Key 本地加密存储（使用系统密钥链或加密文件）
- 所有流量走本地 sqlmapapi，不经过第三方服务器（AI 调用除外）
- 用户可自定义 AI API 地址，支持私有部署

## 10. 附录

### 10.1 术语表

| 术语 | 说明 |
|------|------|
| sqlmapapi | SQLMap 自带的 REST API 服务，通过 `python sqlmapapi.py -s` 启动 |
| Task | sqlmapapi 中的扫描任务单元 |
| Tamper | SQLMap 的绕过脚本，用于绕过 WAF/过滤器 |
| B/E/U/S/T | 五种注入技术：Boolean盲注/Error-based/Union/Stacked/Time盲注 |
| Level | SQLMap 扫描深度等级 (1-5)，越高测试越多 payload |
| Risk | SQLMap 风险等级 (1-3)，越高可能对数据库造成影响 |

|------|------|
