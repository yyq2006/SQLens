## 1. 设计系统

### 1.1 设计风格

| 维度 | 选型 |
|------|------|
| 风格 | **中性实用风** (Neutral Utility) |
| 关键词 | 工具感、清晰、高效、数据密集、中文优先 |
| 参考产品 | Navicat、Postman、Burp Suite Pro、VSCode |
| 设计原则 | 内容优先，操作可达，信息密度适中，减少视觉噪音 |

### 1.3 字体方案

| 用途 | 字体 | 字重 | 字号 |
|------|------|------|------|
| 标题/面板标题 | **Fira Code** | 600 SemiBold | 14px-16px |
| 正文/标签 | **Fira Sans** | 400 Regular | 13px-14px |
| 代码/日志/数据 | **Fira Code** | 400 Regular | 12px-13px |
| 小字/辅助 | **Fira Sans** | 400 Regular | 11px-12px |

> Fira Code 的连字特性对 SQL 语句和代码片段显示非常友好，与"工具感"定位高度匹配。

### 1.4 间距与尺寸

| Token | 值 | 使用场景 |
|-------|-----|---------|
| `space-1` | 4px | 图标与文字间距 |
| `space-2` | 8px | 组件内间距 |
| `space-3` | 12px | 组件间间距 |
| `space-4` | 16px | 面板内边距 |
| `space-5` | 24px | 面板间距 |
| `space-6` | 32px | 大区块间距 |
| `radius` | 6px | 默认圆角 |
| `radius-lg` | 8px | 卡片/弹窗圆角 |

### 1.5 图标风格

| 规则 | 说明 |
|------|------|
| 图标集 | **Lucide Icons** (与 shadcn/ui 默认一致) |
| 大小 | 统一 16px-18px (菜单/工具栏) / 20px-24px (状态图标) |
| 颜色 | 继承文字颜色，状态图标使用语义色 |
| 禁止 | ❌ 不使用 emoji 作为 UI 图标 |

## 3. 组件详细设计

### 3.1 侧边栏 (Sidebar)

```
┌────────────────┐
│ 📂 目标         │  ← 当前选中状态: 主色左侧竖条 + 浅蓝背景
│ 📂 注入         │
│ 📂 检测         │
│ 📂 绕过         │  ← hover: 浅灰背景 #F1F5F9
│ 📂 枚举         │
│ 📂 优化         │
│ 📂 自定义参数    │
│                │
│ ──── 分隔线 ────│
│                │
│ 🎯 历史记录     │  ← 次要导航: 字号略小，颜色 muted
│ 📊 报告         │
│ ⚙️ 设置         │
└────────────────┘

交互:
- 点击导航项 → 滚动到对应参数分组
- 选中态: border-left: 3px solid #0369A1 + bg-blue-50
- 悬停态: bg-gray-50
- 可折叠: 点击左上角汉堡图标隐藏/展开
```

### 3.2 操作栏 (Action Bar)

```
┌─────────────────────────────────────────────────────────────┐
│  🔗  [http://example.com/page?id=1          ]  [▶ 一键全自动] [⏹ 停止] │
│  [模板: 快速检测 ▼]  [导入 ▼]  [导出 ▼]                    │
└─────────────────────────────────────────────────────────────┘

组件:
- URL输入框: 带图标的 Input，支持粘贴自动解析
- 一键全自动按钮: Primary Button (#0369A1)，带 Play 图标
- 停止按钮: Destructive Button (#EF4444)，带 Stop 图标
- 模板下拉: Select 组件
- 导入下拉: DropdownMenu (粘贴URL / 导入文件 / Burp导入)
- 导出下拉: DropdownMenu (导出配置 / 导出结果)

交互:
- 一键全自动 → 触发 AI 全流程，按钮变为 loading 状态
- 模板切换 → 参数面板自动填充对应预设值
```

## 5. 交互流程

### 5.1 一键全自动流程

```
用户打开 SQLens
       │
       ▼
sqlmapapi 自动启动 (后台, 无感)
       │
       ▼
用户粘贴 URL 到输入框
       │
       ▼
点击 [▶ 一键全自动]
       │
       ▼
┌──────────────────────────────────────────────┐
│ ① 按钮变为 Loading 态，不可重复点击          │
│ ② AI 分析请求，状态栏显示 "AI 分析中..."     │
│ ③ 参数面板自动填充 AI 推荐的配置              │
│ ④ 自动创建 Task，启动 sqlmap 扫描            │
│ ⑤ 日志区开始逐条展示扫描日志                  │
│ ⑥ AI 气泡同步解读关键信息                     │
│ ⑦ 发现注入 → 自动继续枚举                     │
│ ⑧ 扫描完成 → 播放完成提示音                  │
│ ⑨ 结果区自动展示数据库结构                    │
│ ⑩ AI 对话窗弹出摘要                           │
└──────────────────────────────────────────────┘
```

### 5.2 自然语言对话流程

```
用户在底部聊天框输入: "扫描这个网站 http://test.com?id=1"
       │
       ▼
AI 识别意图: 扫描请求
       │
       ▼
自动填充参数面板 + 启动扫描
       │
       ▼
AI 回复: "已启动扫描，发现 id 参数存在注入..."

─────────────────────────────────

用户继续输入: "把 users 表数据导出为 CSV"
       │
       ▼
AI 识别意图: 枚举 + 导出
       │
       ▼
自动执行枚举 + 弹出保存对话框
       │
       ▼
AI 回复: "已导出 users 表 (124 行) 到桌面"
```

### 5.3 手动配置扫描流程

```
用户选择模板「快速检测」
       │
       ▼
参数面板自动填充:
  Level=1, Risk=1, 全技术
       │
       ▼
用户手动微调:
  - 取消勾选 Time-based (太慢)
  - 选中 tamper: space2comment
       │
       ▼
点击参数面板顶部的 [▶ 开始扫描]
       │
       ▼
创建 Task → 启动 → 实时日志
```

### 5.4 数据库编辑器流程

```
在数据库浏览器中展开 myapp_db → users
       │
       ▼
点击 [SQL 编辑器] Tab
       │
       ▼
输入: SELECT * FROM users WHERE role='admin'
       │
       ▼
点击 [▶ 执行]
       │
       ▼
通过已发现的注入点发送 SQL 语句
       │
       ▼
结果展示在数据表格中
```
