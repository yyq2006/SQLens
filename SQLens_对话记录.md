## 一、项目背景

### 1.1 初衷

想做一款 **图形化的 SQLMap**，让用户通过 GUI 界面发数据包或输入 URL，进行 SQL 注入分析，而不用写命令行。

### 1.2 定位确认

- ✅ **不是** 修改 SQLMap 源码
- ✅ **不是** Web 端
- ✅ 基于 `sqlmapapi` REST API 做可视化图形外壳
- ✅ 桌面端应用

---

## 二、需求调研过程

### 2.1 竞品调研结论

| 竞品 | 状态 | 核心发现 |
|------|------|---------|
| Havij | ❌ 停更 10 年 | 曾经的 GUI 标杆，已过时 |
| Pangolin | ❌ 2016 停更 | Windows only |
| jSQL Injection | ✅ 活跃 | 独立引擎，Java 桌面，功能全但不如 SQLMap |
| sqlmapWebUI | ✅ 极活跃 | 功能最全的 Web 封装，自帶靶场 |
| SQLI Striker | ✅ 活跃 | 赛博朋克风 Web UI |
| gmh5225/Sqlmap-GUI | ✅ 活跃 | PyQt6 桌面版 |

**竞品核心空白：** 没有引入 AI 的产品，没有全自动化流程的产品。

### 2.2 差异化策略（6 大优势）

| # | 优势 | 说明 |
|---|------|------|
| 1 | 🤖 AI 全流程辅助 | 从分析到报告，AI 全程驱动——竞品均无 |
| 2 | 🖥️ 全参数图形化 | sqlmap 所有常用命令 GUI 化 |
| 3 | 💬 AI 实时解读 | 扫描日志 AI 实时翻译成大白话 |
| 4 | 🛡️ 开发者模式 | 发现漏洞后给出修复建议代码 |
| 5 | 📦 开箱即用 | 单文件打包，自动管理 sqlmapapi |
| 6 | 🔄 工作流集成 | 批量扫描/CI-CD/报告导出 |

## 四、产品方案核心

### 4.1 一句话定位

> **SQLens = 图形化 SQLMap 外壳 + AI 全自动注入分析**

### 4.2 核心架构

```
┌──────────────────────────────────┐
│      SQLens (Electron)           │
│  ┌────────────┐  ┌────────────┐  │
│  │ 渲染进程    │←→│ 主进程      │  │
│  │ React+TW   │IP│ Node.js    │  │
│  └────────────┘  └──────┬─────┘  │
└─────────────────────────┼────────┘
                          │ REST API
              ┌───────────▼──────────┐
              │     sqlmapapi        │
              │  (Python, 自动管理)   │
              └─────────────────────┘
                          │ HTTP
              ┌───────────▼──────────┐
              │  DeepSeek API (在线)  │
              └─────────────────────┘
```

### 4.3 功能模块

1. **sqlmapapi 管理器** — 自动检测/启动/监控/停止
2. **请求编辑器** — URL 输入/数据包粘贴/第三方导入
3. **参数配置面板** — 全部常用参数 GUI 化
4. **一键全自动** — AI 全流程驱动
5. **手动扫描** — 传统方式
6. **实时日志 + AI 解读** — 流式展示 + 智能翻译
7. **数据库浏览器** — 树形浏览/预览/导出
8. **AI 报告** — 自动生成 + 修复建议

### 4.4 迭代路线

| 阶段 | 时间 | 内容 |
|------|------|------|
| P0 MVP | 1 周 | Electron 脚手架 + sqlmapapi 连接 + 基本扫 |
| P1 完整性 | 1 周 | 全参数面板 + AI 集成 + 一键全自动 |
| P2 结果深化 | 1 周 | 数据库浏览器 + AI 报告 + 历史 |
| P3 打磨发布 | 1 周 | 模式切换 + 批量 + 打包发布 |
